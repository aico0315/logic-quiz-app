# ロジック組み立て問題アプリ：全体像

## どんなアプリか

**「こういう処理を書きたい」と思ったとき、実際にコードを書いて動かして確認できる**練習アプリです。

問題例：
> 数値の配列 `[3, 1, 4, 1, 5, 9, 2, 6]` から偶数だけを取り出して `console.log` で出力してください。

ブラウザ上のコードエディタにコードを書いて「実行」ボタンを押すと、実際にコードが動いて出力結果と期待値を比較して正誤判定します。

---

## メソッド問題アプリとの違い

| | メソッド問題アプリ | ロジック組み立て問題アプリ |
|---|---|---|
| 問い | 「このメソッドは何？」 | 「このロジックをコードにすると？」 |
| 回答方法 | メソッド名をテキスト入力 | コードを自由に書いて実行 |
| 判定方法 | 文字列の完全一致 | コードの実行結果と期待値を比較 |

2つのアプリは**あとで統合する予定**のため、同じ技術スタック（React + TypeScript）を使っています。

---

## 技術スタック

| 技術 | 役割 |
|---|---|
| **Vite** | 開発環境のビルドツール |
| **React** | UIコンポーネント管理 |
| **TypeScript** | 型安全なコード |
| **CSS Modules** | コンポーネントごとのスタイル管理 |
| **iframe sandbox** | ユーザーのコードを安全に実行する仕組み |

---

## 画面の流れ

```
メニュー画面（レベル選択）
  ↓ レベルを選ぶ
問題画面（問題表示 → コード入力 → 実行 → 正誤判定・解説表示）
  ↓ 次の問題へ（繰り返し）
クリア画面（スコア表示 → もう一度 or メニューへ）
```

---

## ファイル構成

```
src/
├── main.tsx              # アプリの起動ファイル
├── App.tsx               # 全体の司令塔。画面切り替えを管理
├── types.ts              # TypeScript の型定義
├── index.css             # 全体スタイル
├── lib/
│   └── runner.ts         # コード実行エンジン（このアプリの核心）
├── data/
│   └── questions.ts      # 問題データ
└── components/
    ├── MenuScreen.tsx     # メニュー画面
    ├── QuizScreen.tsx     # 問題・回答・解説画面
    └── ClearScreen.tsx    # クリア画面
```

`lib/runner.ts` がこのアプリで最も重要なファイルです。ユーザーが書いたコードを安全に実行して結果を返す役割を担っています。
